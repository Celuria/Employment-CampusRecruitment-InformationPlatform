## 1. 通用约定

### 1.1 Base URL

| 环境 | Base URL | 说明 |
| --- | --- | --- |
| 本地开发 | `http://localhost:8080/api/v1` | 前端 Vite 代理 `/api` → 后端 |
| 生产环境 | `https://{domain}/api/v1` | Nginx 反向代理 |

### 1.2 RESTful 设计规范

| 规范项 | 约定 |
| --- | --- |
| 风格 | RESTful，资源名使用 **kebab-case** 复数形式 |
| HTTP 方法 | GET 查询、POST 创建、PUT 全量/部分更新、DELETE 删除 |
| 路径参数 | 资源 ID，如 `/career-talks/{id}` |
| 查询参数 | camelCase，如 `?page=1&pageSize=10&keyword=字节` |
| 请求体 | camelCase JSON |
| 响应体 | camelCase JSON |
| 时间格式 | ISO 8601 字符串，如 `2025-01-15T14:00:00+08:00` |
| 日期格式 | `YYYY-MM-DD`，如 `2025-01-15` |
| 布尔值 | `true` / `false` |
| 空值 | 字段不存在或值为 `null` 时不返回该字段（可选优化） |

### 1.3 统一响应结构

所有接口（除文件下载外）均返回以下结构：

```typescript
interface ApiResponse<T = unknown> {
  code: number      // 业务状态码，0 表示成功
  message: string   // 提示信息，成功时为 "success"
  data: T           // 业务数据，失败时可为 null
}
```

**成功示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": { }
}
```

**失败示例：**

```json
{
  "code": 40001,
  "message": "密码长度不能少于 8 位",
  "data": null
}
```

### 1.4 分页约定

#### 请求参数

| 字段 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `page` | `number` | 否 | `1` | 页码，**从 1 开始** |
| `pageSize` | `number` | 否 | `10` | 每页条数，最大 `100` |

#### 响应结构

```typescript
interface PageResult<T> {
  list: T[]         // 当前页数据列表
  total: number     // 符合条件的总记录数
  page: number      // 当前页码
  pageSize: number  // 当前每页条数
}
```

**分页响应示例：**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [],
    "total": 128,
    "page": 1,
    "pageSize": 10
  }
}
```

### 1.5 排序约定

通过查询参数 `sortBy` + `sortOrder` 传递：

| 字段 | 类型 | 可选值 | 说明 |
| --- | --- | --- | --- |
| `sortBy` | `string` | 各接口自定义 | 排序字段 |
| `sortOrder` | `string` | `asc` / `desc` | 排序方向，默认 `asc` |

宣讲会默认：`sortBy=startTime&sortOrder=asc`（按开始时间升序，即将开始的在前）。

### 1.6 HTTP 状态码与业务码关系

| HTTP Status | 典型业务 code | 场景 |
| --- | --- | --- |
| 200 | `0` | 成功 |
| 400 | `400xx` | 参数校验失败 |
| 401 | `40100` | 未登录 / Token 无效 |
| 403 | `40300` | 无权限（角色不足） |
| 404 | `40400` | 资源不存在 |
| 409 | `40900` | 资源冲突（如重复添加日历） |
| 423 | `42300` | 账号锁定 |
| 500 | `50000` | 服务器内部错误 |

> **推荐写法**：HTTP 状态码反映协议层结果，响应体 `code` 反映具体业务错误，便于前端统一拦截处理。

### 1.7 推荐后端 Controller 写法（Spring Boot 示例）

```java
@RestController
@RequestMapping("/api/v1/career-talks")
@RequiredArgsConstructor
public class CareerTalkController {

    @GetMapping
    public ApiResponse<PageResult<CareerTalkVO>> list(CareerTalkQueryDTO query) {
        return ApiResponse.ok(careerTalkService.page(query));
    }

    @GetMapping("/{id}")
    public ApiResponse<CareerTalkVO> detail(@PathVariable Long id) {
        return ApiResponse.ok(careerTalkService.getById(id));
    }
}
```

```java
@Data
public class ApiResponse<T> {
    private int code;
    private String message;
    private T data;

    public static <T> ApiResponse<T> ok(T data) {
        ApiResponse<T> r = new ApiResponse<>();
        r.setCode(0);
        r.setMessage("success");
        r.setData(data);
        return r;
    }

    public static <T> ApiResponse<T> fail(int code, String message) {
        ApiResponse<T> r = new ApiResponse<>();
        r.setCode(code);
        r.setMessage(message);
        return r;
    }
}
```

### 1.8 推荐前端 Axios 封装（已实现）

- 文件：`src/api/request.ts`
- 请求头自动附加：`Authorization: Bearer {token}`
- `code !== 0` 时弹出 `message` 并 reject
- HTTP `401` → 清除 Token，跳转登录页
- HTTP `403` → 提示「无权限」（待实现）

## 2. 认证与鉴权

### 2.1 JWT Token

| 项目 | 说明 |
| --- | --- |
| 传递方式 | Header：`Authorization: Bearer {token}` |
| 有效期 | 建议 2 小时（可配置）；「记住登录」可延长至 7 天 |
| Payload 建议字段 | `sub`(userId), `username`, `role`, `exp`, `iat` |

**JWT Payload 示例：**

```json
{
  "sub": "10001",
  "username": "zhangsan",
  "role": "student",
  "exp": 1735689600,
  "iat": 1735682400
}
```

### 2.2 接口鉴权级别

| 级别 | 说明 | Header 要求 |
| --- | --- | --- |
| 公开 | 访客可访问 | 无 |
| 登录 | 任意已登录用户 | Bearer Token |
| 学生 | 角色为 `student` | Bearer Token |
| 管理员 | 角色为 `admin` | Bearer Token |

### 2.3 权限校验原则

1. **后端强制校验**，前端路由守卫仅做体验优化
2. 学生只能读写 **本人** 的资料、偏好、日历、提醒记录
3. 管理员可 CRUD 校招信息、触发同步、管理用户
4. 管理端写操作需记录 **审计日志**（操作人、时间、动作、资源 ID）

## 5. 认证模块

> 前端文件：`src/api/modules/auth.ts`  
> 鉴权级别：**公开**（登录后返回 Token）

### 5.1 用户注册

**POST** `/auth/register`

#### 请求体 RegisterDTO

| 字段 | 类型 | 必填 | 校验规则 | 说明 |
| --- | --- | --- | --- | --- |
| `username` | `string` | 是 | 4–32 位，字母数字下划线 | 登录账号（学号/邮箱） |
| `password` | `string` | 是 | 长度 ≥ 8，含字母和数字 | 密码 |
| `email` | `string` | 是 | 合法邮箱格式 | 用于邮件提醒 |
| `captcha` | `string` | 否 | — | 验证码（可选） |
| `captchaKey` | `string` | 否 | — | 验证码 Key（可选） |

> 注册仅创建 `role=student` 账号，管理员账号由后台初始化。

#### 响应 data

```typescript
null  // 注册成功无返回体，前端跳转登录页
```

#### 请求示例

```json
{
  "username": "2022001001",
  "password": "Pass1234",
  "email": "zhangsan@edu.cn"
}
```

#### 错误码

| code | message |
| --- | --- |
| `40001` | 用户名已存在 |
| `40002` | 邮箱已被注册 |
| `40003` | 密码不符合复杂度要求 |

---

### 5.2 用户登录

**POST** `/auth/login`

#### 请求体 LoginDTO

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | `string` | 是 | 账号 |
| `password` | `string` | 是 | 密码 |
| `remember` | `boolean` | 否 | 记住登录，默认 `false` |
| `captcha` | `string` | 否 | 验证码（可选） |

#### 响应 data — LoginVO

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `token` | `string` | JWT Token |
| `expiresIn` | `number` | 过期时间（秒） |
| `tokenType` | `string` | 固定 `Bearer` |

#### 响应示例

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 7200,
    "tokenType": "Bearer"
  }
}
```

#### 错误码

| code | message |
| --- | --- |
| `40101` | 账号或密码错误 |
| `42300` | 账号已锁定，请 15 分钟后重试 |

### 11.2 添加到日历

**POST** `/calendar/events`

#### 请求体 CalendarEventCreateDTO

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `eventType` | `EventType` | 是 | 活动类型 |
| `refId` | `number` | 是 | 宣讲会/双选会 ID |
| `customNote` | `string` | 否 | 个人备注 |
| `remindBefore` | `RemindBefore[]` | 否 | 不传则使用用户全局偏好 |

#### 响应 data — CalendarEventVO

返回创建的日历事件。

#### 业务规则

1. 同一用户 + 同一 `eventType` + 同一 `refId` **去重**，重复添加返回 `409`
2. 多个日程时间可重叠，**不做互斥拦截**
3. 添加成功后自动创建邮件提醒任务

#### 错误码

| code | message |
| --- | --- |
| `40901` | 该活动已在您的日历中 |
| `40402` | 关联活动不存在 |

#### 请求示例

```json
{
  "eventType": "career_talk",
  "refId": 1,
  "customNote": "带好简历",
  "remindBefore": ["1d", "1h"]
}
```
